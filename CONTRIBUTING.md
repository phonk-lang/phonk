# Contributing to Phonk

Thank you for your interest in contributing to Phonk!

## Getting Started

1. Clone the repository.
2. Create a new branch.
3. Make your changes.
4. Ensure the project builds successfully and passes all tests.
5. Open a pull request.

## Branch Naming

Please use the following branch naming conventions:

### Features

```
feature/<description>
```

Examples:

```
feature/functions
feature/imports
feature/type-system
```

### Bug Fixes

```
fix/<description>
```

Examples:

```
fix/parser-precedence
fix/lexer-comments
```

### Refactors

```
refactor/<description>
```

Examples:

```
refactor/source-location
refactor/compiler-class
```

### Documentation

```
docs/<description>
```

Examples:

```
docs/contributing-guide
docs/architecture
```

### Continuous Integration

```
ci/<description>
```

Examples:

```
ci/build-workflow
ci/add-gtest
```

### Tests

```
test/<description>
```

Examples:

```
test/parser-tests
test/transpiler-tests
```

## Coding Style

* Use modern C++20.
* Prefer clear and readable code over clever code.
* Write comments if the intent is not obvious.
* Use meaningful names.

## Commit Messages

Please use descriptive commit messages when possible.

Examples:

- feat(parser): add function declarations
- fix(lexer): correct line tracking
- refactor(source): introduce SourceLocation
- test(parser): add operator precedence tests
- ci: add build workflow
- chore: update MIT license

## Pull Requests

Before opening a pull request:

* Ensure the project builds successfully.
* Ensure all tests pass.
* Provide a clear description of what was changed and why.

## Questions

If you have questions, feel free to open an issue or ask in the Discord server.

Happy coding!
